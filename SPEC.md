# 基于 LangGraph 与 Playwright 的爬虫智能体架构

## 1. 概述 (Project Overview)

本规范旨在构建一个能够自主导航、评估网页价值并提取结构化**教育领域用户诉求**的爬虫智能体。系统采用 LangGraph 作为编排框架，通过状态机管理任务流，并利用 Playwright 作为底层浏览器引擎。

## 2. 图架构与节点定义 (Graph Architecture)

智能体采用有状态的循环图结构，将任务拆解为以下核心节点：

### 2.1 核心节点 (Core Nodes)

1. **初始规划节点 (Planner):** 识别目标域名，并生成初始 URL 种子列表 。

2. **信息收集节点 (Judge):**
* **输入:** 当前页面内容。
* **逻辑:** 评估页面是否包含目标数据：
    - 如果有，按照 json 模板解析，并调用保存数据的工具，跳转到页面发现节点。
    - 如果没有，直接跳转到页面发现节点。
* **工具**：[**Redis接口**]

3. **页面发现节点 (Discoverer):**
* **输入:** 当前页面内容。
* **逻辑:** 调用工具点击翻页、进入详情页或构造新的 URL 模式，跳转到信息收集节点。
- 模型不能访问目标域名以外的 URL
* **工具**：[**Playwright**]

4. **自愈节点 (Healer):** 当定位器失效或页面加载超时时，该节点分析错误并重试。

## 3. 工具

1. **Playwright**：浏览器操作

2. **Redis接口**：进行持久化存储、去重。
